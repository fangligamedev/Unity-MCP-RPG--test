# 9 Kings 风格单王国 Alpha 核心循环 SDD

## 1. 文档目标
这份文档定义玩家在首版 Alpha 中会经历怎样的一局游戏，重点是：
- 玩家体验
- 系统行为
- 年度节奏
- 界面与交互
- 首版做什么、不做什么

本文件不出现代码与技术实现细节，只定义玩法结构。

## 2. 产品定位
### 2.1 游戏类型
本项目是一个：
- 单机
- 鼠标优先
- 单局 20 到 40 分钟
- 高重开率
- 回合制构筑 + 自动战斗 + 王国城建
的策略 roguelike 原型。

### 2.2 还原目标
首版目标不是借灵感，而是高还原 9 Kings 的核心体验：
- 以年份推进为节奏单位
- 用卡牌构筑王国版图
- 用自动战斗结算构筑质量
- 用战后抢牌与事件做中长期 build 方向

### 2.3 首版规模
- 玩家可选王国只有 1 个：King of Nothing 风格王国
- 运行中会遇到 2 个外部敌对王国 + 1 个 rebel mirror 敌军
- 框架必须允许后续直接接入第 2、第 3 个玩家王国，而不推翻结构

## 3. 核心体验目标
玩家在一局里应该持续感受到：
- 我在一块有限的王国土地上做艰难但清晰的长期规划
- 每年我都在决定：铺版图、升级、赌战后拿牌、还是把坏牌换成钱
- 每一次敌王来袭都在改变我下一年的构筑方向
- 我的 Build 会越来越离谱，直到 33 年前形成一个自洽而强力的自动战争机器

## 4. 版图与战场
### 4.1 基本结构
- 王国版图最大为 5x5
- 开局只启用中央 3x3，共 9 个 Plot
- 后续最多扩展到 25 个 Plot
- 每个 Plot 是一个独立的内容槽位，同时也是战斗中的逻辑节点

### 4.2 Plot 的承载内容
每个 Plot 需要承载这些信息：
- 坐标
- 是否已解锁
- 当前主卡类型
- 当前等级
- 战斗统计摘要
- 相邻影响摘要
- 当前临时状态，例如战斗生成物、被标记、被祝福目标等

### 4.3 战斗发生方式
- 战斗发生在同一张王国地图上，不切场
- 敌军从边界或预定入口刷新并推进
- Base 位于玩家版图中的一个核心 Plot 上，承担失命判定
- Troop 从所属 Plot 出战
- Tower 保持固定位置攻击
- Building 不一定直接攻击，但会提供战前、年初、战中或相邻增益

### 4.4 路径和压迫感
本项目首版中，敌军应当存在清晰压迫路径：
- 从地图边缘或外圈入口进入
- 以 Base 为最终目标
- 沿着王国边缘与内部通路压进

这样玩家对地块布局、塔位、近战密度和 Buff 邻接才会有真实权衡。

## 5. 年度流程
### 5.1 标准年份流程
1. 年开始，显示当前年份、剩余生命、本年来袭王国、下一个固定事件。
2. 若本年有特殊事件，则先进入该事件界面。
3. 玩家处理手牌：打牌、升级、附魔、施放 Tome、或弃牌进井。
4. 当手牌只剩 2 张时，自动锁定出牌阶段并进入战斗。
5. 自动战斗开始，直到一方胜负成立。
6. 结算：生命变化、金币、战后选卡、统计摘要。
7. 进入下一年。

### 5.2 Final Battle
- 第 33 年为最终战
- 玩家必须把剩余手牌全部处理掉，不能像平常年份那样保留 2 张
- 敌方由该敌王的 champion 领军
- 若最终战中失去生命，整局直接失败
- 若获胜，则视为通关 Alpha 主循环

## 6. 年度事件设计
### 6.1 固定年份事件
首版严格采用固定年份事件表：
- 第 4、14、21、29 年：Royal Council
- 第 6 年：Blessing reveal
- 第 16 年：Blessing effect
- 第 8 年：Diplomat War
- 第 23 年：Diplomat Peace
- 第 10、25、31 年：Merchant
- 第 12、19、27 年：Tower 扩地
- 第 33 年：Final Battle

### 6.2 事件目标
Royal Council
- 用于确定本局 build 方向
- 给永久加成
- 强化重复可叠加的设计乐趣

Blessing
- 用于制造未来数年的地块规划压力
- 让玩家为了第 16 年结果，提前保留或布置特定格子

Diplomat War
- 让玩家主动指定想掠夺哪类敌王卡池
- 本质是战利品方向控制器

Diplomat Peace
- 让玩家剔除不想继续打的敌王来源
- 本质是降低卡池噪音与战斗风格风险

Merchant
- 给金币提供高价值消耗口
- 提供 build 修正与临时找牌能力

Tower 扩地
- 把版图成长绑定到固定节奏上
- 让 3x3、4x4 过渡和 5x5 完成成为一局中的明确成长曲线

## 7. 卡牌交互
### 7.1 打牌
玩家通过拖拽完成以下交互：
- 拖到空 Plot：放置 Base、Troop、Tower、Building
- 拖到同类 Plot：升级
- 拖到已有 Troop：施加 Enchantment
- 拖到已有 Plot：触发 Tome 效果
- 拖到井中：弃牌换钱

### 7.2 手牌规则
- 平常年份必须打牌或弃牌直到只剩 2 张
- 最终战年份必须处理掉全部手牌
- 玩家不能跳过出牌阶段直接战斗

### 7.3 战后选卡
- 战后固定从 3 张候选中选 1 张
- 胜利时卡来自被击败敌王牌池
- 失败时卡来自自己王国牌池
- 允许重掷，但会消耗金币并递增成本

## 8. 自动战斗模式
### 8.1 基本原则
- 玩家不直接控制单位移动、攻击或释放技能
- 战斗完全由版图状态、单位状态和卡牌效果自动解析
- 战斗是对本年构筑质量的检验，而不是补操作阶段

### 8.2 单位与设施职责
Troop
- 负责正面接战、冲线、承伤、输出
- 是 Enchantment 和 Building 邻接增益的主要承载对象

Tower
- 固定在原 Plot 输出
- 负责远程单体或区域火力
- 是防线的稳定器

Building
- 负责年初成长、相邻增益、战斗时触发、资源转换或特殊连锁
- 通常不承担正面主输出

Base
- 既是王国核心，也是失命判定点
- 首版允许保留 1 个可主动触发的特殊能力按钮，但默认也支持自动释放方案，避免操作负担过高

### 8.3 首版战斗抽象程度
首版不追求 9 Kings 的全部单位种类和所有复杂战场演算，而追求：
- 战斗结果可信
- Build 差异明显
- Troop、Tower、Building 各自有存在意义
- 敌军推进、触 Base 失命、清场胜利这三件事清晰成立

## 9. UI 与信息展示
### 9.1 主界面必须展示
- 当前年份
- 剩余生命
- 当前金币
- 当前来袭敌王
- 当前版图 3x3 到 5x5 状态
- 当前手牌
- 井
- Base 状态

### 9.2 Plot 信息层
鼠标悬停 Plot 时，必须能看到：
- 当前卡名与等级
- 核心数值
- 相邻增益来源
- 战斗统计摘要，例如造成伤害、击杀、承伤

### 9.3 事件界面
首版至少需要：
- Royal Council 三选一界面
- Blessing 提示与目标格高亮
- Merchant 商店与重掷界面
- Diplomat War/Peace 选择界面
- Tower 扩地选择界面
- 战后选卡界面

## 10. 首版内容边界
### 10.1 必做
- 33 年完整主循环
- Final Battle
- 3x3 到 5x5 版图成长
- 1 个完整玩家王国
- 2 个敌对王国配置集
- 自动战斗
- 战后选卡
- Royal Council
- Blessing
- Merchant
- Diplomat War/Peace
- Tower 扩地
- 基础统计面板
- 重开与通关失败判定

### 10.2 延后
- 完整 9 王国
- 王国等级 XP 解锁
- 彩虹王令与 broken 王令全量池
- Endless 模式
- 难度梯度
- 成就、排行榜
- 高保真演出特效
- 完整 meta progression

## 11. 成功标准
当玩家在首版 Alpha 中能做到以下几件事时，核心循环就算成立：
- 开局理解自己要先下 Base 再扩王国
- 在前 10 年体会到手牌管理、井和金币的重要性
- 在第 14 到第 21 年形成明显 build 方向
- 在第 25 年之后感到商人、王令和跨王国拿牌正在改变自己构筑的形状
- 在第 33 年前拥有一个明显不同于开局的小型自动战争机器

## 12. 当前设计结论
这个项目的首版不是动作策略混合物，而是一个纯策略自动战斗原型。它的最小成立条件不是做复杂敌兵 AI，而是把这五件事先做对：
- 年份循环
- Plot 版图
- 六类卡牌
- 自动战斗结算
- 战后与年事件节奏
