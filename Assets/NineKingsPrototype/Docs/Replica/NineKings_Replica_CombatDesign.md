# 9 Kings 复刻审阅包：战斗设计文档

## 1. 文档目标
本文件定义 `NineKingsPrototype` 后续要复刻的 9 Kings 式战斗系统，重点补齐以下问题：
- 建筑与兵力如何真实显示在格子场景与战斗场景中
- 自动战斗如何从出牌阶段自然切入
- 单位、建筑、武器与特技如何在同一镜头内可读
- 战斗如何结束、如何切到战利品页

本文件不包含代码，只定义设计规则与审阅标准。

## 2. 参考与事实边界
### 2.1 公开事实来源
- 官方 Wiki：`Cards / Kings / Plots / Run`
- 公开视频与截图：`https://www.youtube.com/watch?v=_YTt9vDATYI`
- 你提供的 1-7 张截图

### 2.2 本文件中的事实分层
- **公开可确认事实**：牌名、王国、基础卡池、出牌-战斗-战利品主循环、控速按钮位置、场景与棋盘关系
- **基于视频与截图的设计推断**：单位具体数量显示方式、某些法术/附魔的可视反馈、近战推进节奏
- **后续实现待拟合项**：精确 DPS、射程帧数、隐藏公式、每张牌的最终数值成长

## 3. 战斗系统的核心目标
玩家必须感受到：
- 我把建筑和部队真实摆在棋盘上，而不是把卡牌丢进一个抽象数值池
- 当手里只剩 2 张牌时，我的布局会直接进入实战检验
- 战斗是“同景自动战斗”，不是切换到另一个独立战斗地图
- 每个建筑、兵种、塔楼和法术强化都能在场景里看到它的存在感

## 4. 战斗主循环
### 4.1 出牌到战斗的切换
固定流程：
1. 玩家在 `Card Phase` 拖拽卡牌到格子、升级、附魔或弃井
2. 当手牌剩余 `2` 张时，出牌阶段锁定
3. 进入 `BattleDeploy`
4. 棋盘上的驻场对象切到战斗预备态
5. 敌军从场景外圈入口生成
6. 进入 `BattleRun`
7. 战斗结束进入 `BattleResolve`
8. 背景压暗，切入 `LootChoice`

### 4.2 为什么必须拆出 BattleDeploy
`BattleDeploy` 不是纯技术细节，而是观感关键阶段：
- 让玩家知道“我刚刚摆下的东西现在要参战了”
- 让建筑、单位、附魔从静态布局态切到动态战斗态
- 给战斗开始前一个短暂但清晰的视觉准备窗口

## 5. 场景内建筑与兵力落位规则
### 5.1 建筑落位
- 建筑永远以世界实体方式落在棋盘格中心
- Base、Tower、Building 都是“棋盘上的世界对象”，不是 HUD 图标
- 建筑必须有：
  - 底座阴影
  - 朝向规则
  - 占格清晰度
  - 与格线的视觉关系

### 5.2 兵力落位
- 战斗单位不替代建筑，而是从建筑或所属格出战
- 友军单位生成后，位置会稍微偏离格中心，避免与建筑重叠
- 敌军从外圈或边界入口出现，推进向 Base
- 小体量单位允许用编队/群点方式表现，而不是每个都必须独立高清角色

### 5.3 数量显示原则
兵力数量不能只写成一个数字，而要同时满足两层表达：
- **主表达**：场景中的战斗单位/编队/群体
- **辅表达**：必要时用浮层数字、堆叠标识、队列感表达数量

推荐规则：
- 1-3：显示独立个体
- 4-8：显示 3 个主个体 + 队列影子或缩略编队
- 9+：显示小编队群体 + 角标数字

## 6. 自动战斗规则
### 6.1 友军
友军分三类：
- 驻场核心：`Palace / Castle`
- 驻场建筑：`Vault / Beacon / Farm / Blacksmith`
- 可参战单位：`Thief / Mercenary / Soldier / Archer / Paladin`
- 塔楼类：`Dispenser / Scout Tower`

### 6.2 敌军
敌军分为通用战斗 archetype：
- 近战推进型
- 远程压制型
- 快速突进型
- 精英耐久型
- Boss / Champion 型

本轮文档只定义显示与行为模板，不伪造完整敌方全量数值。

### 6.3 战斗地图规则
- 战斗仍发生在同一张主战场场景
- 友军占据棋盘区域
- 敌军从外围进入并压向中心
- 右上控速按钮始终可见
- 底部手牌带在战斗阶段收起或弱化，但不彻底破坏同镜头连续性

## 7. 武器、攻击方式与特技反馈
### 7.1 近战
近战单位必须有：
- 向目标冲线或贴近
- 命中瞬间的短促伤害反馈
- 不依赖大量文字说明的清晰接敌感

适用：
- `Thief`
- `Mercenary`
- `Soldier`
- `Paladin`

### 7.2 远程
远程单位和塔楼必须至少具备：
- 出手前的蓄力或朝向提示
- 可见的投射物或射线
- 命中后的伤害反馈

适用：
- `Dispenser`
- `Scout Tower`
- `Archer`

### 7.3 Base / Palace 特技
根据你提供的视频段落，`Palace` 作为 Greed 王国核心，必须具备“明显高于普通 troop 的技能存在感”。

设计要求：
- `Palace` 的主动或自动技能表现应是强辨识的处决/射线/高价值斩杀反馈
- 每次成功触发时必须伴随金币、掠夺或资源收益的视觉提示

### 7.4 附魔与即时牌反馈
`Midas Touch / Over-invest / Mortgage` 等不一定生成单位，但不能“施放后什么都没发生”。

必须定义的表现：
- 目标高亮
- 金色或财富向视觉特效
- 持续态附着框或临时特效
- 如果是经济转化或风险转化，应伴随资源数值变动反馈

## 8. King of Greed 战斗设计结论
### 8.1 这套王国的战斗气质
Greed 王国的核心不是传统重装军团，而是：
- 资源驱动
- 高金币转化
- 建筑与雇佣兵并重
- 刺客/盗贼/雇佣军的效率型打击
- 宫殿与财富建筑的联动

### 8.2 视觉关键词
- 金色、琥珀色、铜色
- 奢华但危险
- 小体量高效率部队
- 资源与战斗互相转化

### 8.3 战斗上应如何看起来像 Greed
- 不是铺满重装大军
- 而是少量关键建筑 + 精准 troop + 高价值技能反馈
- 金币变化、雇佣单位、掠夺收益是主反馈

## 9. King of Nothing 兼容战斗结论
Nothing 王国作为兼容第二王国，其气质与 Greed 必须明显区分：
- Greed：财富、建筑经济、金色视觉
- Nothing：贫瘠、朴素、基础兵种、灰蓝土色视觉

战斗上：
- Nothing 更像基础兵种与防线堆叠
- Greed 更像资源换效率与高价值打击

## 10. 战斗结束规则
### 10.1 胜利
- 敌军清空后进入 `BattleResolve`
- 友军停止推进
- 背景开始暗场处理
- 显示胜利文案
- 进入三选一战利品

### 10.2 失败
- Base 触发失败条件后进入 `BattleResolve`
- 世界仍保留失败战场
- 切入失败结算或失败补牌/失命逻辑

### 10.3 清场
- 残兵不应该瞬间硬消失
- 建议通过冻结、淡出或停战姿态过渡到结算页

## 11. 战后战利品页与战斗的关系
战利品页不是独立 UI 弹窗，而是战斗系统的最后一幕：
- 背景保留战场
- 战场压暗
- 卡牌浮于中景
- 标题与结果信息上浮
- 玩家依然处于“这一局战斗的连续镜头”中

## 12. 审阅结论标准
当这份文档通过时，应满足：
- 你能明确知道建筑和兵力如何在场景中事实显示
- 你能明确知道战斗从出牌如何切入
- 你能明确知道武器、特技和数量不是只靠文本表达
- 你能明确知道 Greed 与 Nothing 两个王国的战斗气质差异
